This page uses avatar components

The component used here is much closer to a standardised library. It allows a much wider gamut of customizations, designed for use across multiple projects with exact use cases not set in concrete. This allows a number of customizations, including:

  • Providing an image for use as an avatar, with a default as well as an option to hide the image entirely.
  • Choosing to show a description if it exists, as well as providing the ability to project a custom description using content projection.

This is how a long lived library component is created, as they're designed to be used in an ongoing manner. There are two approaches to building these components:

  • Adding more features as needed to the component in example #2, which adds complexity over time as backwards compatibility for existing uses has to be maintained.
  • Having a standardised design spec that is being targetted rather than a specific use case. This process has a longer lead up time, but iterating design and implementation in the context of standard features rather than ad-hoc, current usage based additions leads to more flexibility and less maintanence issues in the long term.
